리뷰어가 “무엇이 가능해졌는지” 빠르게 확인할 수 있도록 기능 중심으로 정리했습니다.

PR #2 · Kafka 기반 에이전트 큐

Hangfire/Redis 없이 agent job 실행

  • POST /api/tickets가 Kafka topic에 job 메시지를 발행
  • KafkaAgentWorker가 메시지를 consume하고 AgentJob 실행
  • AgentQueueMessage v1 JSON contract와 round-trip 테스트 추가
  • /health가 DB, Kafka, Docker 상태를 확인
  • Compose에 Kafka dual listener 구성
PR #3 · Provider 분리

운영 환경에 맞는 저장소/알림 선택

  • Remote repository provider: GitHub, GitLab
  • Notifier provider: Slack, Gmail, None
  • Coding agent provider: ClaudeCode
  • GitHub PR 또는 GitLab MR 생성 script 분리
  • Slack이 아니어도 approval을 처리하는 수동 승인/거절 API 추가
PR #4 · 이슈/문서 연동

티켓을 외부 업무 도구와 연결

  • Issue tracker provider: Jira, Linear, None
  • Document tool provider: Notion, Confluence, None
  • 티켓 생성 시 외부 이슈 생성 또는 기존 이슈 reference 연결
  • agent 완료/실패 결과를 외부 이슈 댓글로 남김
  • POST /api/tickets/{id}/documents로 티켓 문서 생성
PR #5 · Telemetry

필요할 때만 켜는 OTLP 계측

  • Telemetry:Enabled로 trace/metric export 활성화
  • ASP.NET Core, HttpClient, runtime metric 수집
  • agent job 시작/완료/실패 counter와 duration 기록
  • 공통 DB 설정 helper로 migration 적용 여부 정리
  • OpenTelemetry 관련 취약 패키지 경고 제거

기능 매트릭스

최종 상태에서 지원되는 provider와 주요 API를 한 번에 볼 수 있습니다.

영역 지원 기능 설정/엔드포인트
Queue Kafka topic 기반 agent job enqueue/consume Queue:KafkaBootstrapServers, Queue:KafkaTopic
Agent execution Docker 컨테이너에서 repo clone, branch 생성, coding agent 실행, PR/MR URL 추출 Agent:ClaudeImage, Agent:MaxConcurrentAgents
Remote repository GitHub PR 생성 또는 GitLab MR 생성 Agent:RemoteRepositoryProvider
Notifier Slack 메시지/버튼, Gmail 메일 알림, No-op 로컬 모드 Notifier:Provider, /api/approvals/{id}/approve, /api/approvals/{id}/reject
Issue tracker Jira/Linear 이슈 생성, 기존 이슈 연결, 완료/실패 댓글 IssueTracker:Provider, CreateTicketRequest.createExternalIssue
Document tool Notion/Confluence 티켓 문서 생성 DocumentTool:Provider, POST /api/tickets/{id}/documents
Observability Serilog 파일/콘솔 로그, secret redaction, 선택형 OTLP trace/metric Telemetry:Enabled, Telemetry:OtlpEndpoint
Health check PostgreSQL, Kafka metadata, Docker daemon 상태 확인 GET /health
