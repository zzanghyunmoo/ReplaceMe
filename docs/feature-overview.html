End-to-end 흐름

가장 중요한 흐름은 티켓 생성부터 agent 실행, 승인 대기, 로그 저장, 완료 알림까지 이어지는 하나의 수직 slice입니다.

1. Ticket`POST /api/tickets`로 요구사항 저장
2. QueueHangfire `agents` 큐에 job 등록
3. AgentDocker 안에서 `claude -p` 실행
4. Approval민감 작업은 Slack 버튼으로 승인
5. Result로그, 상태, PR URL 저장

지금 바로 확인하는 명령

프로젝트 루트에서 다음 순서로 실행하면 로컬 스택을 띄울 수 있습니다.

  • cp .env.example .env
  • docker compose --profile build-only build agent-image
  • docker compose up --build api postgres redis
  • open http://localhost:8080/hangfire
